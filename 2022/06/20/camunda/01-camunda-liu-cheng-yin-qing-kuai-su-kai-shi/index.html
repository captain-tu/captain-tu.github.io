Camunda Quick Start

无标签
Camunda
发布日期:   2022-06-20
文章字数:   1.4k

本教程将指导您完成使用Camunda平台建模和运行您的第一个工作流程。在本指南中，将使用Camunda的现成任务客户端Java，实现可执行流程。

更多关于流程引擎及Camunda的信息请前往官方手册

流程引擎服务搭建及流程建模器下载

首先，您需要安装Camunda平台和Camunda Modeler。
前提条件：Java 运行时环境 1.8+，若使用SpringBoot方式集成开发同时必须包含Java开发环境（JDK） 1.8+

流程引擎服务搭建

安装Camunda流程服务平台有多种方式：Docker方式运行、Camunda官方发行版、SpringBoot集成。本教程将使用最后一种方式，使用SpringBoot嵌入式集成Camunda流程引擎。这种方式可定制化开发，利于企业级应用程序开发。

配置文件

在项目resource目录下新建application.properties文件，添加以下内容

server.port=8081
spring.application.name=process-engine
server.servlet.context-path=/

spring.datasource.type=com.alibaba.druid.pool.DruidDataSource
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
spring.datasource.url=jdbc:mysql://{your-datasource-ip}:{your-datasource-port}/{your-database}?allowMultiQueries=true&useUnicode=true&characterEncoding=UTF-8
spring.datasource.username={your-datasource-username}
spring.datasource.password={your-datasource-password}

# 流程引擎控制台登录账号与密码
camunda.bpm.admin-user.id=camunda
camunda.bpm.admin-user.password=camunda

启动类

新建启动类并启动项目

package org.example.process.engine;

import org.camunda.bpm.spring.boot.starter.annotation.EnableProcessApplication;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

/**
 * @author tufei
 */
@SpringBootApplication
@EnableProcessApplication
public class ProcessEngineApplication {
    public static void main(String[] args) {
        SpringApplication.run(ProcessEngineApplication.class, args);
    }
}

项目启动控制台

流程引擎服务访问，打开浏览器输入 http:localhost:8081，接着输入用户名 camunda 与密码 camunda
