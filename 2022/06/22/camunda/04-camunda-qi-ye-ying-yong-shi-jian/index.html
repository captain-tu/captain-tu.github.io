Camunda 企业应用实践

无标签
Camunda
发布日期:   2022-06-22
文章字数:   2.3k

本篇主要介绍 Camunda 流程引擎 在企业级流程应用开发中的集成与扩展，包括流程核心操作：流程发起、流程同意、流程驳回、流程会签等常规用户操作，以及流程部署、制度文档、审批历史、安全验证等其它功能。在本篇中笔者将会描述流程引擎的集成项目以及核心扩展 API。

工作流引擎 概览

工作流

工作流（Workflow），是对工作流程及其各操作步骤之间业务规则的抽象的、概括的描述。工作流将一套大的业务逻辑分解成业务逻辑段，并统一控制这些业务逻辑段的执行条件，执行顺序以及相互通信，实现业务逻辑的分解和解耦。

在企业日常的管理中，经常会有出差申请、加班申请、请假申请等流程。如果用人工的方式管理这些流程，效率低，管理成本高。因此为了提高效率，我们需要使用到工作流开发。在工作流开发中，我们可以通过每一个流程定义清楚的看到业务的所有步骤，以及每一个流程实例运行到什么位置，参与者是谁；能够节省更多的时间成本，提升企业信息化水平。

Camunda 是一种工作流引擎，是由 Java 开发的一个纯 Java 库。工作流引擎是用来开发工作流的框架。市面上主流的工作流引擎有 Activiti、Flowable、Camunda 等。

  • Activiti 由 Alfresco 公司开发，目前最高版本为 Activiti cloud 7.1.0。由于团队分歧 ，在 Activiti 6 版本衍生出了 Flowable。
  • Flowable 是基于 Activiti 6 衍生出来的版本，开发团队也是 Activiti 中分裂出来的，修复了 Activiti 6 中的 bug，并再此基础上实现的 DMN、BPEL 支持。
  • Camunda 是基于 Activiti 5 的，最新版本是 Camunda 8，开发团队也是从 Activiti 中分裂出来的，发展轨迹和 Flowable 相似。通过压力测试验证 Camunda BPMN 引擎性能和稳定性更好，功能完善；除了 BPMN，Camunda 还支持 CMMN( 案例管理)和 DMN (决策自动化)。Camunda 不仅带有引擎，还有很多强大的工具，用于建模、任务管理、监控和用户管理等。

Camunda

  • Camunda BPM（业务流程管理）平台，用来管理，部署的流程定义、执行任务、策略等等。
  • Camunda Modeler（用于编辑流程图及其他模型）平台，用来定义流程图，简单说就是一个画图工具。

注：关于更多 Camunda 的相关知识，请前往 官方社区 查阅文档。

企业级流程引擎 概览

构建基础

企业级流程引擎服务 基于 Camunda 工作流引擎构建。在官方提供的构建方式中，结合实际需求和应用场景，平台流程引擎服务采用基于 SpringBoot 的构建方式进行嵌入式集成，该方式不仅集成简单、构建方便，同时也能很好地对流程引擎服务进行更多定制化的扩展开发，能够很好的适应和满足不同场景下的业务需求。

  • Camunda：v7.15
  • SpringBoot：v2.4.5
  • Sweet：v3.1.1.1 (内部框架)

企业级流程引擎服务

平台 流程引擎服务 目前涉及 平台 的项目有三个：

  • flow
    • flow-engine
    • flow-engine-sdk
  • config

其中 flow-engine 为 核心服务，对 Camunda 的集成扩展在该部分实现；flow-engine-sdk 为对外服务，主要针对 flow-engine 中的核心服务接口 进行 SDK 封装；config 为 平台 配置中心服务，其中包含了流程制度文档的管理操作。本文对 平台 流程引擎服务 的介绍将 主要围绕 flow-engine 中基于 Camunda 实现和扩展的流程核心操作 展开叙述。

项目介绍

  • bpm 基于 财务共享 对接集团 BPM 的相关操作
  • config 配置类：包括配置 MybatisPlus、Amazon3、Interceptor 等
  • constant 公共常量封装
  • delegates 委托类，在 BPMN2.0 Service Task 类型的任务中进行调用
  • exception 统一异常封装
  • fssc.service 弃用
  • listener 监听类，在 BPMN2.0 User Task 类型的任务中进行调用
  • sign 验签加密封装
  • util 通用工具封装
  • workflow 流程核心，扩展封装
  • EpsFlowEngineApplication 启动类

验签拦截器

验签拦截器：SignInterceptor

平台 流程引擎服务 验签 功能可以保障数据安全，禁止非法访问。 统一采用 RSA 非对称加密方式，提供单例模式获取密钥对，支持【公钥加密/私钥解密】、【私钥加密/公钥解密】两种加解密方式。默认验签实现了基于【私钥加密/公钥解密】的加解密方式，验签参数置入 HTTP Header 中，如下：

  • timestamp = 当前时间戳
  • tenantId = 租户ID
  • processDefinitionKey = 流程编码
  • sign = {timestamp}&{tenantId}&{processDefinitionKey} 加密后的字符串

配置项（Apollo）：

# 是否开启验签安全认证
workflow.sign.enable = true
# 验签过期时间，单位秒
workflow.sign.expireTime = 60

文件服务配置

文件服务配置：CustomAmazonS3Configuration

平台 流程引擎服务 文件配置主要用于用户 预览流程制度文件

配置项（Apollo）：

storage.type = AmazonS3
storage.amazon.access-key = {accessKey}
storage.amazon.secret-key = {secretKey}
storage.amazon.bucket-name = {bucketName}
storage.amazon.service-endpoint = {endPoint}
storage.amazon.job.environment.name = {envName}

邮件服务配置

平台 流程引擎服务 邮件服务主要用于流程事件开始或结束时给目标用户 发送通知邮件

配置项（Apollo）：

spring.mail.host = {host}
spring.mail.port = {port}
spring.mail.username = {username}
spring.mail.password = {password}

流程操作 API

平台流程引擎服务 流程操作 API 统一对外暴露 流程引擎操作接口 ，其中包含：

  • 流程保存（草稿）
  • 流程发起
  • 流程操作（同意、驳回、会签、转办等）

流程保存

接口地址：/flow/engine/workflow/save
请求方式：POST
请求数据类型：application/json
响应数据类型：*/*
接口描述：
请求示例：

{
  "businessKey": "业务编码",
  "empNo": "工号",
  "pageType": "页面类型",
  "processDefinitionKey": "流程编码",
  "processTheme": "流程主题",
  "tenantId": "租户ID",
  "version": "流程版本"
}

响应示例:

{
	"code": "响应码",
	"data": "响应数据",
	"message": "响应消息"
}

流程发起

接口地址：/flow/engine/workflow/start
请求方式：POST
请求数据类型：application/json
响应数据类型：*/*
接口描述:
请求示例:

{
  "businessKey": "业务编码",
  "empNo": "工号",
  "pageType": "页面类型",
  "processDefinitionKey": "流程编码",
  "processTheme": "流程主题",
  "tenantId": "租户ID",
  "version": "流程版本",
  "variables": {流程变量}
}

响应示例:

{
	"code": "响应码",
	"data": "响应数据",
	"message": "响应消息"
}

流程操作

接口地址：/flow/engine/workflow/operate
请求方式：POST
请求数据类型：application/json
响应数据类型：*/*
接口描述：
请求示例：

{
  "businessKey": "业务编码",
  "comment": "操作意见",
  "currentActivityId": "当前实例活动ID",
  "empNo": "工号",
  "extra": "额外字段-会签、转办时指定审批人",
  "notifyType": "通知类型",
  "operateCode": "操作码",
  "processId": "流程实例ID",
  "targetActivityId": "目标实例活动ID-驳回时指定节点",
  "variables": {流程变量}
}

响应示例:

{
	"code": "响应码",
	"data": "响应数据",
	"message": "响应消息"
